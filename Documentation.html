#Creating a Documentation

This documentation is aimed at people who want to document their stuff by using the FSharp.Formatting formatter. First I'll give a little introduction on how it works and where to find what. Also different smaller things like including a graph or an image is shortly explained. If you just want to look those things up feel free to skip to that part by using the drop-down menue.

##Introduction

The libraries of the CSBiology organization are built using the [ProjectScaffold](https://github.com/fsprojects/ProjectScaffold). Besides many other features, this includes automatic Documentation via the [FSharpFormattin](https://github.com/fsprojects/FSharp.Formatting) formatter. This formatter is called when one executes the build.cmd which is located in the project folder. Opening this build.cmd does not only compile the project, it also formats the <b>.fsx</b>(FSharp script file) documents to <b>.html</b> documents by using the FSharpFormatting formatter, besides other things. 
The script files which are formatted have to be placed in "..<i>project name</i>\docs\content". The formatted html files are being placed in "..<i>project name</i>\docs\output". Keep in mind that by using the build.cmd, those files are built and stored only locally.
